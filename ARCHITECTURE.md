# 系统架构

工作组智能体分为四个主要领域，以确保关注点分离和可扩展性。

## 1. 核心层 (`core/`)

智能体的“大脑”和“神经系统”。

- **evolution/**: 自我改进和架构变化的日志。
- **skills/**: 智能体可以使用的可执行脚本或函数定义。
- **config/**: 系统范围的配置文件（例如环境常量）。

## 2. 记忆层 (`memory/`)

智能体的状态存储。

- **context/**: “工作内存”。这里的文件（如 `work_context.md`）经常更新，以反映当前的关注点。
- **archives/**: “长期存储”。完成的任务和历史日志被移到这里。

## 3. 提示词库 (`prompts/`)

智能体的指令集。

- **system/**: 核心身份和行为定义（例如特定的角色）。
- **tasks/**: 经常性工作流的可重复使用模板（例如“周报生成”、“代码审查”）。

## 4. 输出层 (`outputs/`)

“交付区”。智能体为用户生成的所有最终产物都放在这里，以保持根目录整洁。
