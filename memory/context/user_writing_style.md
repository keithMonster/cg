# 用户写作风格与汇报规范 (User Writing Style Guide)

基于您的周报、月报及工作总结历史数据，我为您抽象并总结了以下写作风格与汇报规范。在未来的文档生成中，我将严格遵循此风格。

## 1. 核心基调 (Core Tone)

- **"管理者+工程师" 双重视角**：语气既具备管理者的宏观视野（项目进度、资源协调、价值量化），又不失工程师的严谨细节（版本号、具体技术栈、Bug 原因）。
- **行动导向 (Action-Oriented)**：强调"做了什么"和"结果如何"。避免空泛的描述，使用强有力的动词。
- **客观冷静**：叙述平实、客观，少用形容词堆砌，多用数据和事实说话。
- **成长驱动**：习惯在汇报中体现"技术探索"与"个人/团队成长"，展现持续学习的态度。

## 2. 结构模式 (Structure Patterns)

### 2.1 宏观排版

- **分层级结构**：
  - **一级分类**：通常按"维度"分类（如：核心产品迭代、技术基建、团队管理、跨部门协作）。
  - **二级归纳**：每条记录前使用 **加粗关键词/短语** 进行总结，后接详细描述。
- **核心关键词 (Key Tags)**：在章节开头习惯总结 "Core Keywords"（核心关键词），快速对齐上下文。
  - _例_：`**核心关键词**：哈轴项目、Rolldown-Vite、Notes 1.4`

### 2.2 微观句式

- **"加粗前缀 + 详情阐述" 模式**：
  - _Standard_: `- **[项目/模块名]**：[动词] + [具体事项] + [结果/状态]。`
  - _Example_: `- **CG Notes**：优化智能体截断问题，提升首屏加载速度。`
- **"问题 -> 方案" 闭环模式**：
  - 涉及问题处理时，习惯点出 _原因_ 和 _解决方案_。
  - _Example_: `针对 [问题]，排查发现 [原因]，通过 [方案] 解决。`

## 3. 常用词汇库 (Vocabulary)

### 3.1 动作动词 (High-Frequency Verbs)

- **推进/推动 (Push/Advance)**：用于项目进展（"推进哈轴部署"）。
- **落地 (Land/Implement)**：用于完成交付（"落地看板驱动开发模式"）。
- **攻坚 (Tackle/Storm)**：用于困难任务（"版本攻坚"）。
- **排查 (Troubleshoot)**：用于 Bug 处理（"排查并定位了..."）。
- **重构 (Refactor)**：用于技术优化（"重构通用登录体系"）。
- **梳理 (Sort out/Organize)**：用于流程/文档（"梳理 PRD 文档"）。
- **复盘 (Review)**：用于总结（"组织迭代复盘"）。

### 3.2 专有名词与格式 (Terminology)

- **版本号**：精确到小版本（e.g., `1.6.2`, `1.4.0`）。
- **时间点**：精确到日（e.g., `9月11日`）。
- **产品代号**：使用英文/缩写（e.g., `CG Notes`, `CG Flow`, `QMS`, `PMO Machine`）。

## 4. 内容侧重 (Content Focus)

1.  **产品与业务 (60%)**：
    - 强调 _Showcase_、_评审_、_上线_ 等关键节点。
    - 关注用户反馈与体验细节（如"选人组件回显"、"骨架屏"）。
2.  **技术与基建 (25%)**：
    - 关注架构升级（`Vite`, `BFF`）、性能优化（`Token`, `渲染速度`）。
    - 关注 AI 工程化（`Prompt`, `RAG`, `Agent`）。
3.  **管理与团队 (10%)**：
    - **One-on-One**：不仅记录谈话，还记录 _问题_ 和 _引导策略_。
    - **SOP**：强调标准化的建立（流程、模板）。
4.  **学习与探索 (5%)**：
    - 阅读论文/书籍（《R-Zero》, 《马斯克传》）。
    - 新技术调研（`Embedding`, `Attention`）。

**Summary for AI**: When writing for User, act as a **Lead Architect & Manager**. Be precise, bold key outcomes, use specific version numbers/dates, and always highlight the _value_ of the technical work.
