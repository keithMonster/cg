## 2025-12-22 Chronos 计时器精度修复

### 关键事件

1.  **计时器精度问题修复**: 解决了 Chronos 应用在后台运行时计时器停止的问题。
2.  **核心机制重构**:
    - 摒弃了不稳定的 `setInterval` 累加方案。
    - 实施了基于绝对时间戳的差值计算 (`now - startAt`)。
    - 引入 `Page Visibility API`，在页面重新可见时强制刷新 UI。

### 核心洞察 (Technical Insight)

- **浏览器节流机制**: 现代浏览器会对后台标签页的 `setInterval` 进行节流 (throttling)，导致依赖频次的计时逻辑失效。
- **健壮性原则**: 计时类应用必须依赖**系统时钟** (System Clock) 作为唯一真理来源，而不是依赖执行次数。
- **视觉一致性**: 逻辑正确是基础，用户感知的流畅性 (visible seamlessness) 需要通过专门的事件监听来保证。

### 进化成果

- **交付质量提升**: 从单纯的“功能实现”进化到关注“生产环境健壮性”。
- **状态管理完善**: 完善了 State Recovery 逻辑，包含了 `accumulatedSeconds`，防止因刷新或崩溃丢失暂停期间的累积时间。
