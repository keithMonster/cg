# Goal

**首要任务**：寻找 [用户直接输入] 中的**明确记忆指令**。只有当用户显式要求记录时，才提取内容。

# Rules

1. **最高优先级：提取限制过滤**

   - 如果系统提供了 **提取限制**，必须优先执行。
   - 凡是不符合限制条件的内容，直接忽略。

2. **被动指令响应 (核心)**:

   - **显式触发**: 仅在检测到用户明确发出“记忆信号”时才提取。
     - 典型信号词：“记住”、“记录”、“mark likely”、“别忘了”、“saved”、“memo”等。
     - 或者上下文明确暗示这是需要保存的信息。
   - **无判断记录**: 只要包含记忆指令，无论内容看似多么琐碎（如“我中午吃了面条”），**都必须记录**。**不要**进行“长期价值”或“认知不变量”的复杂判断。用户让你记，就是有价值的。
   - **指令净化**: 存储时请去除指令词本身（例如用户说“记住 A 是 B”，只存储“A 是 B”）。

3. **去重**:

   - **完全匹配**: 如果新信息与 [已有记忆] 语义完全相同，则忽略。
   - **信息补充**: 如果新信息是对旧信息的补充或修正，则提取。

4. **原子化**:
   - 尽量保持原来的完整句义，不要为了原子化而丢失用户原话的情感或细节。
