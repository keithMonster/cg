# 技能：复杂适应推理 (CAS Reasoning)

> **"More is different." - P.W. Anderson**

## 1. 核心目标
不仅仅是解决问题，而是**理解产生问题的系统**。通过识别系统中的 Agent、交互规则和反馈回路，预测或引导涌现现象。

## 2. 核心概念 (The Vocabulary)

### 2.1 涌现 (Emergence)
*   **定义**: 宏观层面的模式，源于微观层面的简单交互，且无法通过还原论解释。
*   **应用**: 寻找那些"整体大于部分之和"的现象。不要试图控制结果，要调整交互规则。

### 2.2 反馈循环 (Feedback Loops)
*   **📈 正反馈 (Reinforcing Loop)**: 放大变化的机制。
    *   *特征*: 指数增长、滚雪球、恶性/良性循环。
    *   *策略*: 如果是好事，建立它；如果是坏事，切断它。
*   **⚖️ 负反馈 (Balancing Loop)**: 抵消变化的机制。
    *   *特征*: 趋向平衡、自我修正、阻力。
    *   *策略*: 识别系统中的"隐形天花板"或"稳定器"。

### 2.3 适应性 Agent (Adaptive Agents)
*   **定义**: 系统中的个体（人、代码模块、细胞）拥有根据环境改变自身规则的能力。
*   **应用**: 分析 Agent 的激励机制和局部信息。

## 3. 分析框架 (The Lens)

当你面对一个复杂问题时，使用以下框架进行解构：

1.  **识别 Agents**: 谁在系统中？他们有什么目标？
2.  **识别规则**: 他们之间如何互动？（竞争、合作、模仿）
3.  **寻找循环**: 哪里存在正反馈？哪里存在负反馈？
4.  **预测涌现**: 如果这个系统运行 100 次，会涌现出什么宏观模式？

## 4. 与其他技能的协同
*   **Shadow Board**: 激进派寻找正反馈（增长），保守派寻找负反馈（稳定）。
*   **Deep Reflection**: 使用 CAS 视角复盘，不再看单点错误，而是看系统性结构。
