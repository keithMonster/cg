# 书籍深度分析工作流

## 执行触发条件
当用户提供书名或要求分析特定书籍时，立即启动此工作流。

## 分析步骤

### 1. 书籍信息收集
- **基础信息**: 书名、作者、出版年份、版本信息
- **背景调研**: 作者背景、写作动机、历史背景
- **影响力评估**: 书籍在相关领域的地位和影响
- **读者反馈**: 主要评价和争议点

### 2. 结构框架分析
- **整体架构**: 章节安排、逻辑脉络、论证结构
- **核心线索**: 主要论点的展开路径
- **关键转折**: 重要的观点转换和深化节点
- **支撑材料**: 案例、数据、引用的分布和作用

### 3. 核心内容提取
- **中心论题**: 书籍要解决的核心问题
- **主要观点**: 作者的核心立场和见解
- **关键概念**: 重要术语和概念的定义
- **方法论**: 作者使用的分析方法和思维框架

### 4. 深层逻辑分析
- **论证逻辑**: 推理过程的严密性和有效性
- **假设前提**: 隐含的基础假设和价值观
- **局限性**: 论证的边界和可能的盲点
- **创新点**: 相对于前人研究的突破和贡献

### 5. 知识价值评估
- **理论价值**: 对学科发展的贡献
- **实践价值**: 对现实问题的指导意义
- **时代意义**: 在当前语境下的相关性
- **学习价值**: 对读者能力提升的帮助

## 输出格式

```

## 一、书籍概览
- 基础信息
- 作者背景
- 创作背景

## 二、结构解析
- 整体框架
- 章节要点
- 逻辑脉络

## 三、核心内容
- 中心论题
- 主要观点
- 关键概念

## 四、深层分析
- 论证逻辑
- 方法论
- 创新贡献

## 五、价值评估
- 学术价值
- 实践意义
- 学习建议
```

## 质量标准
- **准确性**: 确保对书籍内容的理解准确无误
- **深度性**: 不仅停留在表面，要挖掘深层含义
- **系统性**: 形成完整的分析框架
- **客观性**: 平衡展现优点和局限性
