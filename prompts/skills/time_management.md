# 时间管理技能 (Time Management Skill)

## 1. 核心功能

### 1.1 统一时间源原则
- **唯一时间源**: 所有时间获取必须通过系统命令 `date` 动态获取
- **禁止硬编码**: 严禁使用硬编码的日期或时间
- **实时获取**: 每次需要时间信息时都重新获取，确保准确性

### 1.2 标准时间格式
- **文件命名**: `YYYY-MM-DD_HH-MM-SS` (如: 2025-01-15_14-30-25)
- **日期标识**: `YYYY-MM-DD` (如: 2025-01-15)
- **完整时间戳**: `YYYY-MM-DD HH:MM:SS` (如: 2025-01-15 14:30:25)
- **ISO格式**: `YYYY-MM-DDTHH:MM:SS` (如: 2025-01-15T14:30:25)

## 2. 时间获取接口

### 2.1 标准时间获取命令
```bash
# 文件命名时间戳
date +%Y-%m-%d_%H-%M-%S

# 日期标识
date +%Y-%m-%d

# 完整时间戳
date +"%Y-%m-%d %H:%M:%S"

# ISO格式时间
date +"%Y-%m-%dT%H:%M:%S"

# Unix时间戳
date +%s

# 年份
date +%Y

# 月份
date +%m

# 日期
date +%d
```

### 2.2 使用规范
1. **实时获取**: 每次需要时间时都执行相应的date命令
2. **格式选择**: 根据使用场景选择合适的时间格式
3. **一致性**: 同一操作中的所有时间信息应使用同一次获取的结果

## 3. 集成使用示例

### 3.1 文件创建场景
```bash
# 获取时间戳用于文件命名
TIMESTAMP=$(date +%Y-%m-%d_%H-%M-%S)
echo "创建文件: /outputs/report_${TIMESTAMP}.md"
```

### 3.2 记忆管理场景
```bash
# 对话记录时间戳
CONV_TIME=$(date +%Y-%m-%d_%H-%M-%S)
echo "保存对话: /memory/conversations/${CONV_TIME}.md"
```

### 3.3 版本控制场景
```bash
# 获取当前日期用于版本标记
VERSION_DATE=$(date +%Y-%m-%d)
echo "版本日期: ${VERSION_DATE}"
```

### 3.4 日志记录场景
```bash
# 详细时间戳用于日志
LOG_TIME=$(date +"%Y-%m-%d %H:%M:%S")
echo "[${LOG_TIME}] 操作完成"
```

## 4. 最佳实践

### 4.1 时间获取原则
- **即时获取**: 需要时间信息时立即执行date命令
- **单次使用**: 一次操作中的多个时间需求应使用同一次获取的结果
- **格式统一**: 同类操作使用相同的时间格式

### 4.2 常见使用模式
```bash
# 模式1: 单次获取多次使用
CURRENT_DATE=$(date +%Y-%m-%d)
CURRENT_TIME=$(date +"%Y-%m-%d %H:%M:%S")

# 模式2: 组合时间信息
FILE_TIMESTAMP=$(date +%Y-%m-%d_%H-%M-%S)
DISPLAY_TIME=$(date +"%Y年%m月%d日 %H:%M:%S")

# 模式3: 条件时间获取
if [ -n "$NEED_TIMESTAMP" ]; then
    TIMESTAMP=$(date +%s)
fi
```
